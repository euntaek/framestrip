## Why

Simple animations are easy to describe — "200ms ease-out slide from the left" and AI gets it right. But real-world UI motion is often harder to put into words. A page load where elements stagger in with different timings and animations. A multi-step interaction with layered transitions. Something you saw on a website and want to recreate but can't quite articulate.

Instead of trying to describe it, just show it. FrameStrip captures the animation as a frame sequence that you can paste directly into AI.

**Before FrameStrip:**
1. Screen record with QuickTime/OBS
2. Install ffmpeg
3. Extract frames with ffmpeg commands
4. Manually attach images to AI

**With FrameStrip:**
1. Select a region
2. Record
3. Done — paste the frames into AI

## Features

**Capture**
- **Region capture** — capture only the component you care about, not the entire screen
- **Multi-monitor** — select and move regions across any connected display
- **Timed intervals** — 0.1s to 10s between frames, configurable
- **Change detection** — skip identical frames automatically
- **Auto-stop** — set limits by frame count or duration to stop recording automatically

**Interaction tracking**
- **Interaction capture** — automatically capture frames on mouse click/drag with event metadata (position, button, modifiers) saved to `session.json`
- **Cursor display** — optionally show the mouse cursor in captured frames

**Output**
- **AI prompt generation** — one-click copy of a customizable, ready-to-paste prompt with capture metadata
- **Image format** — PNG or JPEG with adjustable quality

**Workflow**
- **Global hotkey** — start recording or show the control panel from anywhere (default: Option+Shift+5, customizable)
- **Live preview** — see captured frames in the menu bar during recording
- **Korean / English** — full localization with system language detection
- **Lightweight** — lives in the menu bar, no ffmpeg or external tools needed

## Installation

### Download

Download the latest `.dmg` from [GitHub Releases](https://github.com/euntaek/framestrip/releases).

Open the `.dmg`, drag FrameStrip to Applications, and launch. On first launch, macOS will ask for screen recording permission.

### Build from Source

Requires Xcode 15+ and macOS 14+ (Sonoma).

```bash
git clone https://github.com/euntaek/framestrip.git
cd framestrip
cp Config/Signing.local.example.xcconfig Config/Signing.local.xcconfig
# Edit Config/Signing.local.xcconfig and set your Apple Developer Team ID
xcodebuild -project FrameStrip.xcodeproj -scheme FrameStrip -configuration Release build
```

The built app will be in `DerivedData/Build/Products/Release/`.

## Usage

### Basic Workflow

1. Click the FrameStrip icon in the menu bar
2. Select **"Select Region & Start Recording"**
3. Drag to select the area containing the animation
4. Adjust the region if needed (resize handles, drag to move)
5. Configure capture interval and options in the control panel
6. Click **Record** (or press Enter)
7. When the animation completes, click **Stop**
8. Click **Copy Prompt** on the completion panel
9. Paste into Claude Code, Codex, or your preferred AI

### Example Prompts

The prompt below is auto-generated by FrameStrip and copied to your clipboard. Just paste it into AI along with the captured frames.

```
<ui-motion-reference>
  <source-material>
  Analyze the frame sequence in @"~/framestrip/framestrip_20260407_143022".
  Treat the saved images as the primary source of visual truth for layout, appearance, visible state, and motion.
  Use @"~/framestrip/framestrip_20260407_143022/session.json" as the authoritative source for timing, capture settings, frame segments, and recorded interaction events.
  Read the `_legend` field in session.json for field descriptions.
  </source-material>

  <analysis-rules>
  Analyze the sequence as time-based motion, not just as a set of static keyframes.
  Pay attention to both macro motion and micro-motion across consecutive frames.

  Explicitly inspect:
  - layout, geometry, spacing, and layering
  - timing, pacing, delays, and easing
  - interaction-triggered state changes
  - persistent ambient motion while the overall composition appears stable
  - per-element changes in opacity, brightness, color, blur, scale, position, distortion, or phase

  If `settings.interactionCapture` is `true`, use recorded interaction events only to infer user intent and trigger UI state changes.
  If `settings.interactionCapture` is `false`, do not assume pointer or keyboard events beyond what is visually evident in the frames.

  When a behavior is uncertain because of capture interval, missing frames, compression, or ambiguity, state that uncertainty explicitly.
  Do not present inference as direct observation.
  </analysis-rules>

  <output-boundaries>
  Reproduce or describe the product UI behavior, not the capture artifact itself.
  Do not recreate a synthetic mouse cursor, click indicator, drag overlay, capture flash, or recording artifact unless it is clearly part of the actual product UI.

  Preserve meaningful ongoing motion even when the screen appears visually stable.
  Do not treat shimmer, flicker, pulsing, twinkling, breathing, or similar subtle motion as decorative noise if it is visible in the frames.
  </output-boundaries>
</ui-motion-reference>

Implement this animation in a single HTML file.
```

Add your own instruction at the end to tell AI what to do — for example:

- `Implement this animation in a single HTML file.`
- `Implement this page transition using Framer Motion with React.`
- `Recreate this loading animation in SwiftUI with spring timing.`
- `Reproduce this button hover transition using CSS transitions.`
- `Analyze this motion in detail before suggesting how to implement it.`
- `What is this kind of animation called? Give me the common name and typical use cases.`

### Settings

Settings are organized in a sidebar layout (Capture / General / Prompt / About):

- **Capture** — interval, image format, change detection, auto-stop limits
- **General** — language (Korean/English), save folder, keyboard shortcut
- **Prompt** — customize the AI prompt template with variable chips
- **About** — version, license, links

### Keyboard Shortcut

Default: `Option + Shift + 5` — starts region selection or shows the control panel during recording. Customizable in Settings.

## Known Limitations

- **Frame timing is not guaranteed at high speeds.** At fast intervals (under 0.5s) with large capture regions (e.g. 4K full screen), the save pipeline may not keep up and some frames will be dropped. This depends on your Mac's CPU and disk speed. PNG is slower to encode than JPEG, so for fast captures, use JPEG or reduce the region size. The completion panel shows the actual number of saved frames.
